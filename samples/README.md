# Тестовые изображения, на которых алгоритм выглядит хорошо

Текущий алгоритм строит рельеф из **яркости/цвета** фото (а не из настоящей глубины),
поэтому лучше всего смотрятся **плавные, контрастные, низкочастотные** картинки с мягкими
цветовыми переходами.

| Файл (загружать в систему) | Что это |
|---|---|
| `input_island.png` | **Остров** — лучший пример: суша поднимается из плоского моря. |
| `input_hills.png`  | Зелёные холмы — мягкий рельеф. |
| `input_dunes.png`  | Песчаные дюны — плавные волны. |

## Как посмотреть результат

1. `dotnet run --project Диплом.csproj`, открыть http://localhost:5000
2. Загрузить любой `input_*.png` → **Generate 3D Level from Photo**, покрутить мышью.

## Почему именно такие картинки работают

- Рельеф = яркость + цвет: плавные переходы → плавные холмы; мелкая текстура реального фото → «рваный» рельеф.
- Контраст даёт высоту; крупные однотонные области → меньше мусорных объектов.
- Зелёный трактуется как «выше», синий (вода) — «ниже», поэтому остров так чётко делится на воду/сушу.

Плохо выглядят: тёмные, низкоконтрастные, сильно детализированные реальные фото.

## Сгенерировать заново / сделать свои

Картинки создаёт `scripts/make_samples.py` (Python + Pillow + numpy), 480×480:

```powershell
python scripts/make_samples.py
```

Он кладёт `sample_*.png` в `wwwroot/uploads/`. Можно менять палитры/частоту и делать свои варианты.
