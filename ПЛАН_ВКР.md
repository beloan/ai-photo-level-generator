# План ВКР

**Тема:** Разработка ИИ-системы генерации игровых уровней на основе реальных фотографий с сохранением игрового баланса

> Структура повторяет образец (4 главы + «Выводы по главе», Глоссарий, Введение, Заключение,
> Список источников, Приложения) и привязана к реально реализованному коду проекта:
> .NET 8 (ASP.NET Core Minimal API), Python-микросервис Depth Anything V2 (FastAPI),
> классический CV на ImageSharp, генерация 2D/3D-уровня, модуль баланса, фронтенд на Three.js,
> экспорт OBJ/GLTF, контейнеризация Docker.

---

## ГЛОССАРИЙ
Краткие определения ключевых терминов (10–15 шт.), например:
- **PCG (процедурная генерация контента)** — автоматическое создание игрового контента по алгоритму/модели.
- **Монокулярная оценка глубины** — восстановление карты глубины по одному RGB-изображению.
- **Depth Anything V2** — предобученная трансформерная модель оценки глубины (DPT-подобная архитектура).
- **Карта глубины (depth map)** — попиксельная оценка относительного расстояния до камеры.
- **Карта высот (heightmap)** — поверхность, высота которой задаётся 2D-массивом значений.
- **Сегментация заливкой (flood fill)** — выделение связных областей по близости цвета/яркости.
- **Оператор Собеля** — свёрточный детектор границ (градиент яркости).
- **Игровой баланс** — свойства уровня: проходимость, кривая сложности, распределение ресурсов.
- **Проходимость (playability)** — существование маршрута от старта к финишу (поиск в ширину, BFS).
- **glTF / OBJ** — форматы обмена 3D-геометрией для игровых движков (Unity и др.).
- **Минимальный API (Minimal API)** — упрощённая модель определения HTTP-эндпоинтов в ASP.NET Core.
- **Инференс** — применение обученной модели к новым данным без дообучения.

## ВВЕДЕНИЕ
- **Актуальность.** Рост спроса на пользовательский контент в играх; дороговизна ручного дизайна
  уровней; доступность фотографий как источника пространственной структуры; прогресс в монокулярной
  оценке глубины (Depth Anything V2) делает реалистичную реконструкцию по одному фото практичной.
- **Проблема.** Большинство PCG-подходов генерируют уровни «из шума»/правил и не опираются на реальную
  пространственную структуру; реконструкции по фото редко доводятся до **играбельного, сбалансированного**
  уровня.
- **Объект исследования** — процесс автоматической генерации игровых уровней.
- **Предмет исследования** — методы и алгоритмы преобразования одиночной фотографии в играбельный 3D/2D
  уровень с проверкой игрового баланса.
- **Цель** — разработать ИИ-систему, которая по реальной фотографии строит игровой уровень с сохранением
  игрового баланса.
- **Задачи:**
  1. Проанализировать предметную область (PCG, монокулярная глубина, CV-анализ сцены, баланс уровней).
  2. Формализовать конвейер «фото → анализ → уровень → проверка баланса».
  3. Реализовать оценку глубины (нейросеть Depth Anything V2 + классический фолбэк по яркости).
  4. Реализовать CV-анализ сцены (границы, сегментация, классификация объектов).
  5. Реализовать построение 2D/3D-уровня и модуль баланса (проходимость, сложность).
  6. Реализовать веб-систему (бэкенд, микросервис, фронтенд-визуализатор, экспорт в движок).
  7. Провести экспериментальное исследование точности и качества генерации.
- **Методы исследования** — компьютерное зрение, глубокое обучение (инференс), процедурная генерация,
  алгоритмы на графах (BFS), программная инженерия (микросервисная архитектура).
- **Научная новизна** — комбинированный конвейер, объединяющий монокулярную нейросетевую глубину,
  классический CV и проверку игрового баланса в единую систему генерации уровней по фото.
- **Практическая значимость** — инструмент для быстрого прототипирования уровней дизайнерами; экспорт
  готовой геометрии в Unity (OBJ/GLTF).
- **Структура работы** — краткое описание глав.

**2.1. Постановка задачи и общий конвейер**
